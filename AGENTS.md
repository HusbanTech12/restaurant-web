# AGENTS.md — Savory Restaurant Web (AI-Native)

## Commands

```bash
npm run dev          # next dev --webpack (Webpack, not Turbopack)
npm run build        # next build --webpack
npm run lint         # eslint (flat config, v9)
npm run db:generate  # drizzle-kit generate
npm run db:migrate   # drizzle-kit migrate
npm run db:push      # drizzle-kit push (dev only)
npm run db:studio    # drizzle-kit studio
npm run db:seed      # tsx src/db/seed.ts
```

No test framework, no typecheck script — run `tsc --noEmit` manually.

## Architecture

### Stack
| Layer | Technology |
|---|---|
| Frontend | Next.js 16.2.9 App Router, React 19, TypeScript (strict), Tailwind v4 |
| Backend | Next.js API Routes (`src/app/api/`) |
| Auth | Clerk |
| ORM | Drizzle |
| DB | Supabase (Postgres) |
| AI | OpenAI SDK, Groq API |
| Email | Resend |

### Directory layout
```
src/
  app/
    api/             # All backend logic lives here (route handlers)
      auth/          # Clerk webhooks
      chat/          # AI agent conversation endpoint
      menu/          # Menu CRUD / search
      reservations/  # Reservation CRUD
      orders/        # Order CRUD
      contact/       # Contact form -> Resend email
    (pages)          # Client components, "use client"
  components/        # Shared UI components
  data/              # Static seed data (menu items, reviews, etc.)
  db/                # Drizzle schema & migrations
    schema.ts        # All DB table definitions
    migrations/      # Generated by drizzle-kit
  lib/               # Utility modules
    ai.ts            # OpenAI / Groq client setup + agent tools
    auth.ts          # Clerk helpers (server-side)
    email.ts         # Resend client
    db.ts            # Drizzle client (Supabase Postgres connection)
```

### Data flow
- **Server-only code**: API routes, Drizzle queries, OpenAI/Groq calls, Resend -- never in client
- **Client components** fetch via `fetch("/api/...")` -- no direct DB or AI SDK calls from browser
- **Clerk** protects API routes via middleware; `auth()` returns userId
- **Static seed data** (`src/data/`) co-exists with DB -- reference data for agents; write to DB for user-generated content

## Rules & Conventions

### API Routes (`src/app/api/`)
- Every route handler wraps logic in try/catch, returns `NextResponse.json({ error })` with proper status
- Use `export const runtime = "edge"` only when needed -- default to Node.js runtime
- Route params: `export async function GET(req: NextRequest, { params }: { params: Promise<{ id: string }> })`

### Auth (Clerk)
- Protect API routes with `auth()` from `@clerk/nextjs/server` -- check `userId` at top of handler
- Public routes (menu listing, AI chat without account) explicitly skip auth check
- Clerk webhooks (user create/update/delete) at `src/app/api/auth/webhooks/route.ts` -- verify `svix` signature
- Client components use `@clerk/nextjs` components (`<SignIn />`, `<SignUp />`, `<UserButton />`)

### Database (Drizzle + Supabase)
- All schema in `src/db/schema.ts` -- use `pgTable`, `serial`, `text`, `timestamp`, `boolean`, `jsonb`
- Every table gets `id`, `createdAt`, `updatedAt` via `drizzle-orm` helpers
- Migrations generated with `drizzle-kit generate`; never edit migration files manually
- Query in API routes only: `import { db } from "@/lib/db"`
- No raw SQL unless absolutely necessary -- use Drizzle query builder

### AI (OpenAI SDK + Groq API)
- API keys in `.env.local`: `OPENAI_API_KEY`, `GROQ_API_KEY`
- All AI calls happen in API routes only -- never expose keys to client
- Use `openai` SDK with structured outputs / tool-calling for agent features
- Groq API for faster inference on smaller models (via `groq-sdk`)
- Agent tools defined in `src/lib/ai.ts` -- each tool wraps a Drizzle query or data lookup

### Email (Resend)
- Send transactional emails from API routes via `resend.emails.send()`
- Templates: React Email components in `src/emails/` or plain HTML strings
- Used for: contact form confirmations, reservation receipts, order confirmations

### Pages (Client)
- All pages are `"use client"` unless explicitly SSR -- no Server Components
- Forms now POST to `/api/*` instead of showing mock success
- Use `useAuth()` from `@clerk/nextjs` for client-side auth state

## Key conventions (UI)

- **Tailwind v4**: `@import "tailwindcss"` (not `@tailwind`). Theme in `@theme inline` in `globals.css`.
- **Animations**: Framer Motion v12 -- `motion.div`, `whileInView`, `AnimatePresence`.
- **Icons**: `lucide-react` (tree-shakeable named imports).
- **Three.js** installed but unused in `src/` -- do not add more unless asked.

## Environment variables

Required in `.env.local`:
```
DATABASE_URL=postgres://...
DIRECT_URL=postgres://...      # Supabase direct connection for migrations
OPENAI_API_KEY=sk-...
GROQ_API_KEY=gsk_...
CLERK_SECRET_KEY=sk_...
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_...
RESEND_API_KEY=re_...
```

## Gotchas

- `fix-swc.sh` disables native SWC binary to prevent SIGBUS on WSL2 -- run if SWC crashes.
- No `prettier` config -- formatting is ad-hoc.
- Lockfile: `package-lock.json` (npm, not pnpm/yarn).
- Drizzle migrations require `DIRECT_URL` (not pooled `DATABASE_URL`) for DDL statements.
- Clerk webhooks must be configured in Clerk Dashboard to point to `https://.../api/auth/webhooks`.
- Seed DB before use: `npm run db:push && npm run db:seed` (requires `.env.local` with valid Supabase credentials).
